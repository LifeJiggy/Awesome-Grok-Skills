This directory contains comprehensive skill files for red team operations, organized into 5 specialized subfolders.

## Subfolder Descriptions

### 1. Penetration Testing
- **GROK.md**: Comprehensive framework for authorized security assessments covering reconnaissance, vulnerability assessment, exploitation, and reporting
- **penetration_testing.py**: Full Python implementation with classes for network scanning, vulnerability assessment, web application testing, and privilege escalation

### 2. Exploit Development
- **GROK.md**: Framework for developing, testing, and deploying exploits for authorized security testing
- **exploit_development.py**: Complete implementation with buffer overflow exploits, ROP chains, web application exploits, and payload generation

### 3. Social Engineering
- **GROK.md**: Framework for conducting authorized human-factor security testing including phishing, vishing, and physical social engineering
- **social_engineering.py**: Full implementation with campaign management, pretext generation, and analytics

### 4. Red Team Operations
- **GROK.md**: Comprehensive framework for full-spectrum adversary simulations and security assessments
- **red_team_operations.py**: Complete implementation with campaign planning, execution, and reporting

### 5. Adversary Emulation
- **GROK.md**: Framework for simulating real-world threat actors and their TTPs
- **adversary_emulation.py**: Full implementation with MITRE ATT&CK integration, TTP implementation, and detection validation

## Features

Each skill file includes:
- **GROK.md**: YAML frontmatter, overview, core capabilities, usage examples, best practices, and related modules
- **Python file**: Module docstring, imports, enums, dataclasses, main engine class, helper classes, and demo function

## Usage

These skill files are designed for authorized security testing and red team operations. They provide:

1. **Comprehensive documentation** for understanding red team methodologies
2. **Full Python implementations** with classes, methods, type hints, and error handling
3. **Demo functions** showing how to use the frameworks
4. **Best practices** for ethical and legal compliance
5. **Integration points** with industry-standard tools and frameworks

## Compliance

All frameworks are designed for authorized security testing only. Users must:
- Obtain explicit written permission before testing
- Follow rules of engagement
- Maintain operational security
- Document all activities
- Comply with legal and ethical standards
