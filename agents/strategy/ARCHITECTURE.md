## 4. Design Patterns

| Pattern | Application |
|---------|-------------|
| **Facade** | `StrategyAgent` unifies seven subsystems |
| **Template Method** | SWOT analysis and BMC use structured templates |
| **Strategy** | Different scoring algorithms per subsystem |
| **Registry** | Competitors, segments, scenarios stored as dicts |
| **Observer** | Risk level changes trigger reclassification |
| **Composite** | Objectives compose Key Results and Initiatives |

| Dimension | Strategy |
|-----------|----------|
| Objective Count | In-memory dict; migrate to DB |
| Competitor Count | Flat dict; add search for large sets |
| Scenario Count | Evaluate lazily on demand |
| Market Segments | Add filtering for large markets |
| Multi-tenant | Add tenant_id to all entities |

## 8. Extension Points

1. **New SWOT Strategy**: Extend `_generate_strategies()` with custom TOWS rules
2. **Custom Risk Scoring**: Override `StrategicRisk.calculate_risk_score()`
3. **Market Research API**: Connect `MarketAnalyzer` to external data sources
4. **BMC Templates**: Add pre-built canvas templates per industry
5. **Dashboard Export**: Add JSON-to-Chart rendering pipeline
